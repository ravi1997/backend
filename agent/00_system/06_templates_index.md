# Templates Index

## Purpose

A central registry for all document and code templates used by the Agent.

## Central Templates (`agent/07_templates/`)

### Specification & Design

- `agent/07_templates/srs/03_functional_requirements.md`: Template for features and user stories.
- `agent/07_templates/srs/04_nonfunctional_requirements.md`: Template for performance, security, and scale.
- `agent/07_templates/diagrams/ERD.mermaid.md`: For database design.
- `agent/07_templates/diagrams/FLOWCHART.mermaid.md`: For logic and process flows.

### Implementation

- `agent/07_templates/feature/feature_kickoff.md`: Template for starting a new developer task.
- `agent/07_templates/feature/implementation_summary.md`: Template for reporting finished work.

### Quality & Governance

- `agent/07_templates/pr/pull_request_template.md`: Format for peer reviews.
- `agent/07_templates/release/release_notes_template.md`: Format for final project delivery.
- `agent/10_security/threat_model_template.md`: Format for analyzing risks.
- `agent/14_known_issues/KI_HARD_CASE_TEMPLATE.md`: Template for documenting hard-case incidents with Issue-Key mapping.

## State Templates (`agent/09_state/`)

- `agent/09_state/PROJECT_STATE.md`: The master lifecycle tracker.
- `agent/09_state/BACKLOG_STATE.md`: The task list tracker.

## Usage Rule

The Agent must ALWAYS copy the raw content of these templates before filling them to ensure no headers or structural elements are missed.
